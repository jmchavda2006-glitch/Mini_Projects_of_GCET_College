# 🛒 Automated Billing Management System

A multi-functional, console-based desktop retail invoicing application built strictly around Object-Oriented Programming (OOP) paradigms. This system interfaces with structured local text streams to manage real-time store inventory balances and transaction states.

## 🛠️ Tech Stack & Core Concepts
* **Language:** Java (JDK 8 or higher)
* **Data Structures:** Java Collections Framework (`ArrayList`, `HashMap`, `Iterator`)
* **Core APIs:** Java File I/O (`BufferedReader`, `FileWriter`), Java Time API (`LocalDateTime`)

## 💡 Core Features
* **Admin Authentication Layer:** Built-in programmatic credential protection cross-referencing keys from a local flat file (`login.txt`).
* **Inventory Handling:** Dynamic CRUD setup (Create, Read, Update, Delete) to track running item names, stock capacities, and pricing scales.
* **Automated Invoice Engine:** Manages a virtual digital shopping cart, flags insufficient inventory volumes, auto-applies a 5% promotional discount, and appends standard 18% GST markups.
* **Formatted Document Generation:** Serializes transaction variables directly into structured text sheets (`.txt`) and auto-opens them on local text viewers (like Notepad).

## 🚀 How to Run Locally

Since this project runs directly as a standalone Java application, no heavy installation steps are required. 

### 1. Prerequisite
Ensure you have the Java Development Kit (JDK 8 or higher) installed and configured in your environment system path.

### 2. Compilation

Open your terminal inside this project directory and compile the program:

```cmd
javac BillingSystem.java

```

### 3. Execution

Run the compiled execution program: 



 Windows / Mac / Linux:

```bash
java BillingSystem

```
